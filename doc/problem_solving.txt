备份文件的程序运转清单
1.需要备份的文件与目录应在一份列表中予以指定。
2.备份必须存储在一个主备份目录中。
3.备份文件将打包压缩成 zip 文件。
4.zip 压缩文件的文件名由当前日期与时间构成。
5.我们使用在任何 GNU/Linux 或 Unix 发行版中都会默认提供的标准 zip 命令进行打包。在这里你需要了解到只要有命令行界面，你就可以使用任何需要用到的压缩或归档命令。

# # 1.更好的文件命名机制
# # 使用时间作为文件名，存储在以当前日期为名字的文件夹中（这一文件夹则照常存储在主备份目录下）
# # 优点：
# # - 1.你的备份会以分层的形式予以存储，从而使得它们能更易于管理
# # - 2.文件名能够更短
# # - 3.由于只有当天进行了备份才会创建相应的目录，独立的目录能够帮助你快速地检查每天是否都进行了备份

遵循我们在编写备份脚本时所经历的步骤：
1.进行分析与设计；
2.开始实现一个简单版本；
3.测试并修复错误；开始使用以确保工作状况皆如期望那般。
现在，你可以添加任何你所希望拥有的功能，并继续去重复这一“开始做—测试—使用”循环，需要做多少次就去做多少次。
